10:T17ec,<h2>Overview</h2>
<p>Hadoop Distributed File System (HDFS) is the primary storage layer used by Hadoop workloads. It is designed for high-throughput data access on commodity hardware and is most effective for write-once, read-many patterns.</p>
<p>HDFS is usually not the right fit for ultra-low-latency random reads.</p>
<h2>Architecture</h2>
<p>HDFS follows a master-slave model with three core roles:</p>
<ul><li>NameNode: manages namespace and metadata.</li><li>DataNode: stores data blocks and serves read/write requests.</li><li>Client: requests metadata from NameNode and streams data with DataNodes.</li></ul>
<h3>NameNode</h3>
<p>The NameNode handles:</p>
<ul><li>File system tree metadata.</li><li>Block mapping and placement.</li><li>Access control and policy checks.</li></ul>
<p>For production, run a high-availability setup to avoid single-point-of-failure risk.</p>
<h3>DataNode</h3>
<p>DataNodes handle:</p>
<ul><li>Block storage.</li><li>Heartbeats to NameNode.</li><li>Block checksum validation.</li></ul>
<h3>Client</h3>
<p>Client responsibilities:</p>
<ul><li>File operations.</li><li>Metadata lookup via NameNode.</li><li>Direct block transfer with DataNodes.</li></ul>
<h2>Data Storage Model</h2>
<h3>Block Size</h3>
<pre><code class="language-bash"># Check block size
hdfs dfsadmin -confKey dfs.blocksize

# Common defaults
# Hadoop 1.x: 64MB
# Hadoop 2.x: 128MB
# Hadoop 3.x: 128MB (can be larger)</code></pre>
<h3>Replication</h3>
<pre><code class="language-bash"># Check replication factor
hdfs dfsadmin -confKey dfs.replication

# Common default: 3</code></pre>
<p>Write path summary:</p>
<ol><li>Client streams to the first DataNode.</li><li>DataNode forwards in pipeline to other replicas.</li><li>Write is acknowledged when replica writes complete.</li></ol>
<h2>Useful HDFS Commands</h2>
<h3>Basic Operations</h3>
<pre><code class="language-bash"># Create directory
hdfs dfs -mkdir /user/hadoop

# Upload file
hdfs dfs -put localfile.txt /user/hadoop/

# Download file
hdfs dfs -get /user/hadoop/remote.txt local.txt

# List files
hdfs dfs -ls /user/hadoop/

# Remove file
hdfs dfs -rm /user/hadoop/oldfile.txt</code></pre>
<h3>Administrative Operations</h3>
<pre><code class="language-bash"># Set replication recursively
hdfs dfs -setrep -R 3 /user/hadoop/

# Disk usage
hdfs dfs -du -h /user/hadoop/

# Cluster report
hdfs dfsadmin -report

# Safe mode
hdfs dfsadmin -safemode enter
hdfs dfsadmin -safemode leave</code></pre>

&lt;/configuration&gt;</code></pre>

<h2>Performance Optimization</h2>
<h3>Block Size Strategy</h3>
<ul><li>Large files: prefer larger block sizes.</li><li>Small files: avoid large volumes of tiny files.</li><li>Align block and processing split strategy for batch jobs.</li></ul>
<pre><code class="language-bash"># Example custom block size on upload
hdfs dfs -D dfs.blocksize=268435456 -put largefile.dat /user/hadoop/</code></pre>
<h3>Memory Tuning Example</h3>
<pre><code class="language-xml">&lt;property&gt;
  &lt;name&gt;mapreduce.map.memory.mb&lt;/name&gt;
  &lt;value&gt;1536&lt;/value&gt;
&lt;/property&gt;
&lt;property&gt;
  &lt;name&gt;mapreduce.reduce.memory.mb&lt;/name&gt;
  &lt;value&gt;3072&lt;/value&gt;
&lt;/property&gt;</code></pre>
<h2>Security Basics</h2>
<h3>Authentication</h3>
<pre><code class="language-bash">hdfs dfs -D hadoop.security.authentication=kerberos -ls /</code></pre>
<h3>Permissions and Ownership</h3>
<pre><code class="language-bash"># Set permissions
hdfs dfs -chmod 750 /user/hadoop/protected

# Set ownership
hdfs dfs -chown hadoop:hadoop /user/hadoop/data

# Verify permissions
hdfs dfs -ls -la /user/hadoop/</code></pre>
<h2>Monitoring and Maintenance</h2>
<h3>Health Checks</h3>
<pre><code class="language-bash"># Cluster health
hdfs dfsadmin -report

# Live DataNodes
hdfs dfsadmin -report -live

# File/block checks
hdfs fsck /user/hadoop -locations -files -blocks</code></pre>
<h3>Maintenance Tasks</h3>
<pre><code class="language-bash"># Roll NameNode edits
hdfs dfsadmin -rollEdits

# Refresh DataNodes
hdfs dfsadmin -refreshNodes

# Decommission workflow
echo &quot;datanode3.example.com&quot; &gt; decommissioning-nodes.txt
hdfs dfsadmin -refreshNodes</code></pre>
<h2>Common Issues</h2>
<h3>NameNode Not Starting</h3>
<pre><code class="language-bash">tail -f /var/hadoop/hdfs/namenode.log
jps | grep NameNode
hdfs dfsadmin -safemode get</code></pre>
<h3>DataNode Connection Problems</h3>
<pre><code class="language-bash">hdfs dfsadmin -report -dead
tail -f /var/hadoop/hdfs/datanode.log
rm -rf /var/hadoop/hdfs/datanode/*
hdfs datanode -format</code></pre>
<h3>Disk Space Pressure</h3>
<pre><code class="language-bash">hdfs dfsadmin -report
hdfs dfs -du / | sort -nr | head -10
hdfs dfs -expunge</code></pre>

<ol><li>Keep large analytical files in HDFS.</li><li>Avoid too many small files.</li><li>Use compression where possible.</li><li>Organize datasets with clear partitioning strategy.</li><li>Monitor
