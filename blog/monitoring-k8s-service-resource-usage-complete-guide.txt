10:T182d,<h2>Overview</h2>
<p>Monitoring Kubernetes resource usage is required for reliability, performance, and cost control. This guide covers baseline checks, deeper analysis, and simple automation scripts for daily operations.</p>
<h2>Basic Resource Monitoring</h2>
<h3><code>kubectl top</code></h3>
<pre><code class="language-bash"># Pod usage
kubectl top pods

# Node usage
kubectl top nodes

# Real-time watch
kubectl top pods --watch

# Namespace scope
kubectl top pods -n namespace-name

# Sort by usage
kubectl top pods --sort-by=memory
kubectl top pods --sort-by=cpu</code></pre>
<h3>Quotas and Limits</h3>
<pre><code class="language-bash"># Namespace quotas
kubectl get quota -n namespace-name

# Limits in namespace
kubectl get limits -n namespace-name

# Pod requests/limits
kubectl get pods -o custom-columns=NAME:.metadata.name,CPU:.spec.containers[*].resources.requests.cpu,MEM:.spec.containers[*].resources.requests.memory,CPULIMIT:.spec.containers[*].resources.limits.cpu,MEMLIMIT:.spec.containers[*].resources.limits.memory</code></pre>
<h2>Advanced Resource Analysis</h2>
<h3>Pod-Level Details</h3>
<pre><code class="language-bash">kubectl get pods -o yaml | grep -E &quot;(resources|requests|limits)&quot;
kubectl top pods --containers</code></pre>
<h3>Node-Level Details</h3>
<pre><code class="language-bash">kubectl describe nodes

kubectl get nodes -o custom-columns=NAME:.metadata.name,ALLOCATABLECPU:.status.allocatable.cpu,ALLOCATABLEMEM:.status.allocatable.memory,ALLOCATEDCPU:.status.capacity.cpu,ALLOCATEDMEM:.status.capacity.memory

kubectl get nodes -o jsonpath=&#039;{.items[*].status.capacity}{&quot;\n&quot;}&#039;
kubectl get nodes -o jsonpath=&#039;{.items[*].status.allocatable}{&quot;\n&quot;}&#039;</code></pre>
<h2>Metrics Server</h2>
<h3>Install and Verify</h3>
<pre><code class="language-bash">kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

kubectl get deployment -n kube-system metrics-server
kubectl get pods -n kube-system

kubectl top nodes
kubectl top pods</code></pre>
<h3>Debug Metrics Server</h3>
<pre><code class="language-bash">kubectl logs -n kube-system deployment/metrics-server
kubectl edit deployment -n kube-system metrics-server</code></pre>
<h2>Prometheus and Grafana</h2>
<h3>Prometheus Config Example</h3>
<pre><code class="language-yaml">apiVersion: v1
kind: ConfigMap
metadata:
  name: prometheus-config
  namespace: monitoring
data:
  prometheus.yml: |
    global:
      scrape_interval: 15s
      evaluation_interval: 15s
    scrape_configs:
      - job_name: kubernetes-pods
        kubernetes_sd_configs:
          - role: pod</code></pre>
<h3>Grafana Quick Setup</h3>
<pre><code class="language-bash">helm repo add grafana https://grafana.github.io/helm-charts
helm repo update
helm install grafana grafana/grafana
kubectl port-forward svc/grafana 3000:80</code></pre>
<h2>Critical Checks</h2>
<h3>Hotspot Detection</h3>
<pre><code class="language-bash">kubectl top pods --sort-by=cpu | head -10
kubectl top pods --sort-by=memory | head -10

kubectl get pods --field-selector=status.phase=Pending</code></pre>
<h3>Efficiency Review</h3>
<pre><code class="language-bash">kubectl top nodes | awk &#039;
{
  cpu_usage = $3
  cpu_total = $4
  mem_usage = $5
  mem_total = $6
  cpu_util = cpu_usage / cpu_total * 100
  mem_util = mem_usage / mem_total * 100
  printf &quot;Node: %s - CPU: %.1f%% Memory: %.1f%%\n&quot;, $1, cpu_util, mem_util
}&#039;</code></pre>
<h2>Resource Governance Best Practices</h2>
<h3>Requests and Limits Example</h3>
<pre><code class="language-yaml">apiVersion: v1
kind: Pod
metadata:
  name: resource-demo
spec:
  containers:
    - name: frontend
      image: nginx
      resources:
        requests:
          memory: &quot;64Mi&quot;
          cpu: &quot;250m&quot;
        limits:
          memory: &quot;128Mi&quot;
          cpu: &quot;500m&quot;
    - name: backend
      image: redis
      resources:
        requests:
          memory: &quot;128Mi&quot;
          cpu: &quot;500m&quot;
        limits:
          memory: &quot;256Mi&quot;
          cpu: &quot;1000m&quot;</code></pre>
<h3>HPA Example</h3>
<pre><code class="language-bash">kubectl autoscale deployment my-app --cpu-percent=70 --min=2 --max=10
kubectl get hpa
kubectl describe hpa my-app</code></pre>
<h2>Troubleshooting</h2>
<pre><code class="language-bash"># Evicted pods
kubectl get pods --all-namespaces | grep Evicted

# Node pressure indicators
kubectl describe node &lt;node-name&gt; | grep -i pressure

# Scheduling failures
kubectl get events --field-selector=reason=FailedScheduling -A</code></pre>
<h2>Automation Scripts</h2>
<h3>Daily Report Script</h3>
<pre><code class="language-bash">#!/bin/bash

echo &quot;=== K8S Resource Usage Report ===&quot;
echo &quot;Date: $(date)&quot;
echo &quot;&quot;

echo &quot;=== Node Resource Usage ===&quot;
kubectl top nodes | column -t
echo &quot;&quot;

echo &quot;=== Top CPU Pods ===&quot;
kubectl top pods --sort-by=cpu | head -10
echo &quot;&quot;

echo &quot;=== Top Memory Pods ===&quot;
kubectl top pods --sort-by=memory | head -10
echo &quot;&quot;

echo &quot;=== Pending Pods ===&quot;
kubectl get pods --field-selector=status.phase=Pending</code></pre>
<h3>Threshold Alert Script</h3>
<pre><code class="language-bash">#!/bin/bash

NAMESPACES=(&quot;production&quot; &quot;staging&quot; &quot;development&quot;)
CPU_THRESHOLD=80
MEMORY_THRESHOLD=85

for namespace in &quot;${NAMESPACES[@]}&quot;; do
  echo &quot;Checking namespace: $namespace&quot;

  kubectl top nodes --no-headers | while read -r name cpu mem rest; do
    cpu_usage=$(echo &quot;$cpu&quot; | sed &#039;s/%//&#039;)
    mem_usage=$(echo &quot;$mem&quot; | sed &#039;s/%//&#039;)

    if (( $(echo &quot;$cpu_usage &gt; $CPU_THRESHOLD&quot; | bc -l) )) || (( $(echo &quot;$mem_usage &gt; $MEMORY_THRESHOLD&quot; | bc -l) )); then
      echo &quot;WARNING: $name CPU=$cpu Memory=$mem&quot;
    fi
  done

done</code></pre>
